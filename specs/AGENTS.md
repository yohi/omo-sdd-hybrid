# specs 知識ベース

## OVERVIEW
仕様駆動開発（SDD）の中心であり、タスク定義および機能仕様（Feature Spec）を管理するディレクトリです。開発の全工程はここから始まります。アーキテクトが仕様を定義し、実装者がそれに基づき作業を行うための基準点となります。

## STRUCTURE
- `tasks.md`: 現在進行中および完了したタスクの管理リスト（Active）。
- `<feature>/requirements.md`: 個別の機能に関する要件・設計ドキュメント（Design）。

## CONVENTIONS
- **タスク記述形式**:
  `* [ ] ID: タイトル (Scope: \`glob\`)`
  （例: `* [ ] Task-1: 認証機能の実装 (Scope: \`src/auth/**\`)`）
- **スコープ指定**:
  有効な glob パターンの指定が必須です。Gatekeeper プラグインはこのスコープに基づきファイルアクセスを制御します。

## ANTI-PATTERNS
- **不適切なスコープ定義**: `Scope: **` や `Scope: src/**` のように広すぎるスコープを定義すること。可能な限り最小限の範囲に絞ってください。
- **実行中のタスク修正**: タスクが `in_progress` 状態のときに、`tasks.md` 内のタスク定義を直接修正すること。変更が必要な場合は、一度タスクを終了させてから修正してください。
